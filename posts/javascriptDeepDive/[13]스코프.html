[13] 스코프

자바스크립트 스코프에 대해 알아봅시다!

#Javascript#모딥다#이론
이 글은 모던 자바스크립트 Deep Dive를 읽고 정리한 글입니다.
오늘은 스코프에 대해서 알아본 내용을 정리해 봅시다.
스코프란 식별자가 자신이 선언된 위치에 의해 다른 코드가 식별자 자신을 참조할 수 있는 유효 범위이다.
스코프란 쉽게 공간을 떠올리면 좋을 것 같습니다.
function add(x, y) {
    return x + y;
}
함수를 선언하고 매개변수를 받아 사용하는 과정이 가장 쉽게 스코프 개념을 접근할 수 있는 것 같습니다.
매개변수 x,y는 add함수 외부에서는 참조 할 수 없습니다.
그림을 보시면 더 이해하기 쉽겠죠?
📌 var의 중복선언
var키워드로 선언한 변수는 같은 스코프 내에서 중복 선언이 가능하다. 이는 의도치 않은 재 할당 오류가 발생할 수 있다.
function foo() {
    var x = 1;
    var x = 2;
    return x; // 2
}

function bar() {
    let x = 1;
    let x = 2; // SyntaxError
    return x;
}

스코프의 종류

코드는 전역과 지역으로 구분할 수 있습니다. 표를 볼까요?
구분설명스코프변수
전역코드 가장 바깥전역 스코프전역변수
지역함수 몸체 내부지역 스코프지역 변수
지역 스코프에 있는 지역 변수는 자신의 지역 스코프와 하위 지역 스코프에서 참조할 수 있습니다.
물론 전역 스코프에 있는 변수는 어디서든지 참조 할 수 있겠죠?

스코프 체인

함수 내부에서 함수가 정의된 것을 함수의 중첩이라 하고, 중첩된 함수를 중첩 함수, 중첩함수를 포함한 함수를 외 부 함수라 부릅니다.
아까 그림에서 **foo()**가 외부 함수, **bar()**가 중첩 함수 이 것을 함수의 중첩이라 할 수 있겠네요!
함수가 중첩되는 것은 스코프 또한 중첩될 수 있다는 것을 말합니다.
이는 스코프가 함수 중첩에 따라 계층적 구조를 갖는다고 할 수 있겠네요.
스코프의 계층적 구조를 스코프 체인이라 하고, 모든 지역 스코프의 최상위 스코프는 전역 스코프 입니다.
스코프 체인은 물리적인 실체로 존재합니다.
자바스크립트 엔진은 코드를 실행하기 앞서 위 그림과 유사한 자료구조인 렉시컬 환경을 생성합니다.
변수 선언이 실행되면 변수 식별자가 렉시컬 환경에 key로 등록 되고, 변수 할당이 일어나면, 이 자료구조의 변수 식별자에 해당하는 값을 변경합니다. 변수의 검색 또한 이 자료구조 상에서 이루어집니다.
📌 스코프 체인은 실행 컨텍스트의 렉시컬 환경을 단 방향으로 연결한 것이다. 전역 렉시컬 환경은 코드가 로드되면 곹바로 생성되고 함수의 렉시컬 환경은 함수가 호출되면 곧바로 생성된다. 이에 대해서는 23장에서 더 살펴보자
📌 렉시컬 환경의 동작 과정은 다음과 같습니다:
  • 변수나 함수가 선언될 때, 해당 식별자와 값을 포함하는 환경 레코드가 생성됩니다.
  • 렉시컬 환경은 외부 렉시컬 환경 참조를 통해 상위 스코프와 연결됩니다.
  • 변수를 참조하면 현재 렉시컬 환경의 환경 레코드에서 식별자를 검색합니다.
  • 만약 식별자를 현재 렉시컬 환경에서 찾으면 해당 값을 반환합니다.
  • 찾지 못한 경우, 외부 렉시컬 환경으로 이동하여 검색을 반복합니다.
  • 이 과정을 외부 렉시컬 환경 참조를 따라 상위 스코프로 이동하면서 반복합니다.
  • 최종적으로 식별자를 찾을 때까지 스코프 체인을 따라 올라갑니다.
(고마워요 ChatGPT)
자바스크립트 엔진은 스코프 체인을 따라 변수를 참조하는 코드의 스코프에서 시작해서 상위 스코프 방향으로 이동하면서 선언된 변수를 검색합니다.
스코프 계층적 구조는 부자관계로 이루어진 상속과 유사하다. 상속을 통해 부모의 자산을 자식이 자유롭게 사용할 수 있지만 자식의 자산을 부모가 사용할 수 없다. (class의 상속을 말한거 겠지만… 현실에 대입하니까 왜캐 슬프지….😭)

함수 레벨 스코프

아까 표에서 지역은 함수 몸채 내부를 말한다고 했던거 기억하시나요?
C나 Java를 비롯한 대부분의 프로그래밍 언어는 함수뿐 아니라 모든 코드 블록 ( 를 사용하는 “문” if, for, while, try,catch)이 지역 스코프를 만듭니다.
이런 특성을 블록 레벨 스코프라고 합니다.
하지만 자바스크립트에서 var키워드로 선언된 변수는 오로지 함수의 코드 블록만을 지역 스코프로 인정합니다.
이러한 특성을 함수 레벨 스코프라고 합니다.
이러한 특성때문에 발생하는 오류가 많습니다… (const,let 만세!)
var i = 10;

for (var i = 0; i < 5; i++);

console.log(i); // 5
아까 var는 같은 스코프 내에서 중복선언을 할 수 있다고 했는데,
이는 for블록 밖에 있는 i와 for안에 선언된 i가 동일한 변수라는 것을 뜻합니다.
즉 블록 안에서 선언된 변수가 전역변수를 재할당 하는 오류를 범할 수 있습니다.
for에서 선언한 변수를 외부에서 사용할 일은 거의 없겠죠?
예시코드에서는 의도치 않게 함수 내부에서 선언한 i가 전역 변수인 i를 덮어쓰는 오류가 발생합니다.
var i = 10;

for (let i = 0; i < 5; i++);

console.log(i); // 5
(let을 쓰면 해결!)

렉시컬 스코프

프로그래밍 언어는 일반적으로 상위 스코프를 결정하는 방식이 2가지가 있습니다.
  • 함수를 “어디서” 호출 했는지에 따라 함수의 상위 스코프를 결정한다.
  • 함수를 “어디서” 정의 했는지에 따라 함수의 상위 스코프를 결정한다.

1. 동적스코프

1 번과 같은 방식을 동적스코프라고 합니다.
함수를 정의하는 시점에는 함수가 어디서 호출될 지 알 수 없기 때문에 함수가 호출되는 시점에 동적으로 상위 스코프를 결정해야 하기 때문에 동적 스코프라 부릅니다.

2. 렉시컬 스코프

2번과 같은 방식을 정적 스코프 또는 렉시컬 스코프라 부릅니다.
동적 스코프 방식처럼 상위 스코프가 동적으로 변하지 않고 함수 정의가 평가되는 시점에 상위 스코프가 정적으로 결정 되기 때문에 정적 스코프라고 부릅니다.
자바스크립트를 비롯한 대부분의 프로그래밍 언어는 렉시컬 스코프를 따른다.
즉, 자바스크립트에서 함수의 상위 스코프는 항상 자신이 정의된 위치에서 상위 스코프를 결정한다.
